What's an API? (And Why Should You Care?)
The concept in plain English — no jargon, no BS.
5 min read Concepts

Think of it like a drive-through window

You know how the reporting UI works — you log in, click through menus, pick a date range, hit export, wait, get a spreadsheet. That's like walking into a restaurant, sitting down, ordering from a menu, and waiting for your food.

An API is the drive-through window. You pull up, say exactly what you want, and get it handed to you. Same kitchen, same food, but faster and you never have to get out of your car.

Why does this matter for you?

  • No UI limitations — the reporting module only shows you certain fields. The API gives you everything.
  • Combine data — remember combining two admin reports into one? The API can pull both and merge them automatically.
  • Automate it — that report you pull every month? Imagine double-clicking a file and having it appear on your desktop, ready to go.
  • Exact data, every time — no clicking the wrong filter, no "oops, wrong date range." The script remembers.

How it works (30 seconds)

Every API interaction has the same pattern:

  1. Authenticate — "Hi, I'm Heather, here are my credentials." (You do this once.)
  2. Request — "Give me the GL detail report for February 2026."
  3. Response — The system hands you the data. CSV, JSON, whatever you asked for.

That's it. Everything else is just details about how to ask. Which is what the next lessons cover.

Installing Postman — Your New Best Friend
Download, install, and take your first look around.
10 min Postman

What is Postman?

Postman is a free app that lets you talk to APIs by filling in forms and clicking buttons. No code. Think of it as a testing tool — you'll use it to build and test your requests before we turn them into scripts later.

Install it

  1. Go to postman.com/downloads
  2. Download the desktop app (not the web version)
  3. Install it — next, next, finish
  4. Create a free account when prompted (use your work email)

Quick tour

When you open Postman, you'll see:

  • Collections (left sidebar) — groups of saved requests. This is where your imported collection will live.
  • Request tab (center) — where you build/edit a request. Has a URL bar, method dropdown (GET/POST), and tabs for headers, body, params.
  • Response panel (bottom) — where the data comes back after you click Send.

Don't worry about memorizing everything. In the next lesson, you'll import a pre-built collection and just start clicking.

Your First API Call — Pulling Real Data
Import the collection, authenticate, and pull your first report.
15 min Postman Hands-on

Import the collection

  1. Go to the Downloads tab on this site
  2. Download "Heather's UKG Ready Collection"
  3. In Postman: click Import (top left) → drag the file in
  4. You'll see a new collection appear in the sidebar with pre-built requests

Set your credentials

The collection uses variables so you don't have to type credentials every time:

  1. Click the collection name in the sidebar
  2. Go to the Variables tab
  3. Fill in your values (Chuck will provide these):
    • base_url — the API server address
    • company_id — your company's ID
    • client_id — your API client ID
    • client_secret — your API client secret
  4. Click Save

Get your token

Before you can pull data, you need to authenticate (remember the drive-through analogy? This is pulling up to the window).

  1. Open the "🔑 Get Auth Token" request in the collection
  2. Click Send
  3. You should see a response with an access_token
  4. The collection automatically saves this token for all other requests — you don't have to copy/paste it

Pull your first report

  1. Open "📊 Get Saved Report (CSV)"
  2. Click Send
  3. Look at the response — that's your data, pulled straight from the API!
  4. Click Save Response → Save as File to download it as CSV

🎉 You just made your first API call. Everything from here builds on this exact pattern: authenticate → request → get data.

Exploring Endpoints — What Else Can I Pull?
Saved reports, employee data, GL details — what's available.
10 min Postman API

Endpoints = different windows at the drive-through

Each endpoint gives you different data. The collection includes the most useful ones for tax and GL work:

Saved Reports

These are the reports already built in the UI. The API can run them and return results as CSV:

GET /ta/rest/v1/report/saved/{report_id}

You'll find the report ID in the URL when you open a saved report in the browser. The collection has your most-used ones pre-loaded.

Employee Data

GET /ta/rest/v2/companies/{company_id}/employees

Full employee records — names, IDs, departments, hire dates, everything. Way more fields than the UI export.

What about filters?

Most endpoints accept query parameters to narrow results:

  • Date ranges: ?start_date=2026-01-01&end_date=2026-01-31
  • Specific employees: ?employee_id=12345
  • Pagination: ?page=1&per_page=100

In Postman, these go in the Params tab — just fill in the key and value. The collection has examples.

PowerShell 101 — Your First Script
From VBA to PowerShell — same thinking, new syntax.
15 min PowerShell

Why PowerShell?

You already think like a scripter — VBA taught you that. PowerShell is the same idea but built for talking to APIs and manipulating data outside of Excel. And it's already on your machine.

VBA → PowerShell translation

Concepts you already know:

  • Variables: VBA: Dim x As String → PS: $x = "hello"
  • If/Then: VBA: If x = 1 Then → PS: if ($x -eq 1) { }
  • Loops: VBA: For Each item In collection → PS: foreach ($item in $collection) { }
  • Output: VBA: Debug.Print → PS: Write-Host

Open PowerShell

  1. Press Win + X → click Windows PowerShell (or Terminal)
  2. Type Write-Host "Hello Heather!" and press Enter
  3. Congrats, you just ran PowerShell 🎉

Your first API call in PowerShell

This is the same thing you did in Postman, but in code:

# Step 1: Authenticate (get a token)
$authBody = @{
    grant_type    = "client_credentials"
    client_id     = "YOUR_CLIENT_ID"
    client_secret = "YOUR_CLIENT_SECRET"
}

$token = Invoke-RestMethod `
    -Uri "https://secure2.saashr.com/ta/rest/v2/companies/YOUR_COMPANY_ID/oauth2/token" `
    -Method POST `
    -Body $authBody

# Step 2: Pull a saved report
$headers = @{ Authorization = "Bearer $($token.access_token)" }

$report = Invoke-RestMethod `
    -Uri "https://secure2.saashr.com/ta/rest/v1/report/saved/YOUR_REPORT_ID" `
    -Headers $headers `
    -Method GET

# Step 3: Save it
$report | Out-File "$env:USERPROFILE\Desktop\my-report.csv"

Write-Host "Done! Check your desktop." -ForegroundColor Green

Save that as Get-MyReport.ps1 on your desktop. Right-click → Run with PowerShell. CSV appears on your desktop. ✨

Making It Yours — Parameters & Customization
Change date ranges, pick different reports, filter results.
10 min PowerShell API

Dynamic dates

Instead of hardcoding dates, let PowerShell figure them out:

# Last month's start and end
$lastMonth = (Get-Date).AddMonths(-1)
$startDate = Get-Date $lastMonth -Day 1 -Format "yyyy-MM-dd"
$endDate = Get-Date (Get-Date $lastMonth -Day 1).AddMonths(1).AddDays(-1) -Format "yyyy-MM-dd"

Write-Host "Pulling data from $startDate to $endDate"

Reusable config

Put your credentials in a separate file so your scripts stay clean:

# config.ps1 — keep this private!
$Config = @{
    BaseUrl      = "https://secure2.saashr.com"
    CompanyId    = "YOUR_COMPANY_ID"
    ClientId     = "YOUR_CLIENT_ID"
    ClientSecret = "YOUR_CLIENT_SECRET"
}

# In your report scripts, just dot-source it:
. .\config.ps1
# Now $Config.BaseUrl, $Config.ClientId etc. are available

Export to Excel-friendly CSV

# If the API returns JSON, convert to CSV properly
$data | ConvertTo-Csv -NoTypeInformation | Out-File "report.csv" -Encoding UTF8

# Or if it's already CSV text from a saved report:
$report | Out-File "report.csv" -Encoding UTF8

# Auto-open in Excel
Start-Process "report.csv"

Now you can modify any script — different report IDs, different date ranges, different output files. You're not copying someone else's work anymore, you're building your own tools.

📋 Quick Reference

Authentication Flow

POST /ta/rest/v2/companies/{company_id}/oauth2/token

Body (form-urlencoded):
  grant_type    = client_credentials
  client_id     = your_client_id
  client_secret = your_client_secret

Response:
  { "access_token": "abc123...", "expires_in": 3600 }

Use in all subsequent requests:
  Header: Authorization: Bearer abc123...

Common Endpoints

# Saved Reports (CSV)
GET /ta/rest/v1/report/saved/{report_id}
Accept: text/csv

# Employee List
GET /ta/rest/v2/companies/{company_id}/employees

# Employee Details
GET /ta/rest/v2/companies/{company_id}/employees/{employee_id}

# Time Entries
GET /ta/rest/v2/companies/{company_id}/time-entries

PowerShell Cheat Sheet

# GET request
$result = Invoke-RestMethod -Uri $url -Headers $headers -Method GET

# POST request with body
$result = Invoke-RestMethod -Uri $url -Method POST -Body $body

# Save to CSV
$result | Export-Csv -Path "output.csv" -NoTypeInformation

# Read a CSV
$data = Import-Csv "input.csv"

# Filter rows
$filtered = $data | Where-Object { $_.Department -eq "Tax" }

# Select columns
$slim = $data | Select-Object Name, EmployeeId, Amount

# Sort
$sorted = $data | Sort-Object Amount -Descending

# Group & summarize (like a pivot table!)
$data | Group-Object Department | ForEach-Object {
    [PSCustomObject]@{
        Department = $_.Name
        Count      = $_.Count
        Total      = ($_.Group | Measure-Object Amount -Sum).Sum
    }
}

Common Status Codes

  • 200 — Success! Data is in the response.
  • 401 — Not authenticated. Token expired? Re-run the auth request.
  • 403 — Forbidden. You don't have permission for this endpoint.
  • 404 — Not found. Check the URL/report ID.
  • 429 — Too many requests. Slow down, wait a few seconds.
  • 500 — Server error. Not your fault. Try again in a minute.

🔧 Setting Up OAuth in UKG Ready

Before you can use the API, you need to create OAuth credentials in UKG Ready. This is a one-time setup. Here's how:

1️⃣
Navigate to API Setup
  1. Log into UKG Ready as an administrator
  2. Go to Menu → System Configuration → Security → OAuth API
  3. You'll see the OAuth API Clients page
2️⃣
Create a New API Client
  1. Click Add to create a new API client
  2. Give it a descriptive name (e.g., "Heather - Tax Reports")
  3. For Grant Type, select Client Credentials
  4. Set the Token Expiration (default 3600 seconds = 1 hour is fine)
  5. Click Save
3️⃣
Copy Your Credentials

After saving, UKG Ready will show you:

  • Client ID — a long string like 960ca8a3-ba97-4910-845c-a18ac718d9fe
  • Client Secret — another long string (⚠️ this is only shown ONCE — copy it now!)

⚠️ Important: The client secret is shown only at creation. If you lose it, you'll need to regenerate it. Save both values somewhere safe (like a password manager).

4️⃣
Set API Permissions
  1. On the API client you just created, go to the Permissions or Scope tab
  2. Enable access to the endpoints you need:
    • Reports — for pulling saved reports
    • Personnel — for employee data
    • Time — for time entries (if needed)
  3. Click Save

💡 Tip: Start with read-only permissions. You can always add more later. Read-only means you can pull data but can't accidentally change anything.

5️⃣
Find Your Company ID

Your Company ID is in the URL when you're logged into UKG Ready. Look for the number after /companies/ in the address bar. It's also visible on the API settings page.
